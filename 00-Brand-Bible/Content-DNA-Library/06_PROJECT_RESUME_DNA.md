# Project Resume DNA

## Purpose
Present a project as credible evidence of work, judgment, and learning without inflating Vikash's role or the project's maturity.

## Best use cases
Build logs, portfolio notes, project retrospectives, launch records, and career-transition evidence.

## Recommended content pillars
Build With Vikash, AI Visibility Journal, and Leadership With Vikash.

## Core narrative structure
Problem -> Vikash's actual role -> stage and constraints -> work completed -> evidence -> learning -> current state.

## Hook patterns
- "I moved from using marketing tools to trying to build one."
- Lead with the real problem or transition, not a title or achievement claim.

## Body development
State whether the project was explored, prototyped, built, tested, launched, or scaled. Describe personal contribution accurately and disclose MVP or experimental status.

## Ending patterns
End with the next honest milestone, open question, or transferable lesson.

## Tone
Specific, modest, evidence-led, and professionally useful.

## Evidence requirements
Repository evidence, working output, test record, approved link, or other verifiable artifact. Role and maturity claims must match the evidence.

## What to avoid
Treating built as launched, tested as proven, launched as scaled, an MVP as a mature product, or AI-assisted work as unsupported senior-engineering expertise.

## Example topics based on genuine experience
- The 2017 move from offline marketing into digital as a documented professional transition.
- QueryCite's origin and current build stage, using only verified capabilities.
- Moving from using marketing tools to building them.

## Quality checklist
- [ ] Role and contribution are accurate.
- [ ] Project stage is explicit.
- [ ] Claims have an artifact or approved evidence.
- [ ] Limitations and next steps remain visible.

## Strong combinations with other DNA
AI Builder for the learning process; Failure and Recovery for setbacks; Candid Operator for a real transition story.
