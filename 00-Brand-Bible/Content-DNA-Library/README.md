# VikashOS Content DNA Library

This is the canonical library for **how** VikashOS content is written. Content pillars and series define **what** Vikash writes about; Content DNA defines the narrative method used to express it. A DNA must never be used to create a duplicate pillar or series.

Start with the event, evidence, or idea. Select one primary DNA, then add one supporting DNA only when it improves the post. Use no more than three DNA types in one post.

## Selection Rules

1. Identify what actually happened and which permanent pillar or series owns the topic.
2. Choose the primary DNA that best carries the central lesson.
3. Add one supporting DNA only if it performs a distinct job.
4. Use a third DNA only in an exceptional case; three is the absolute maximum.
5. Evidence and confidentiality rules override stylistic preference.
6. Do not force a story, joke, controversy, child-related detail, failure, or question.
7. Record primary and supporting DNA in the draft metadata and performance record.

## Integration Audit

The repository and full Git history were searched for all 18 framework names and the original eight names before this library was created. No historical copies of Humor DNA, Kid Wisdom DNA, SaaS Success DNA, Wrap DNA, Step Guide DNA, Project Resume DNA, Paid Ads DNA, or Corporate DNA were present. The existing broad [Content DNA](../CONTENT_DNA.md) and [Candid Operator Storytelling](../CANDID_OPERATOR_STORYTELLING.md) were found and remain canonical for identity, pillars, voice, and the full story-led standard. This library operationalises the approved framework definitions without creating new pillars or duplicating those source documents.
