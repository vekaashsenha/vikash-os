# LinkedIn System

This folder manages the public publishing side of VikashOS.

LinkedIn is where practical learning becomes public thinking. Posts should not be treated as isolated updates. Each post should support the long-term narrative: Vikash Sinha is evolving from an experienced B2B marketing leader into an AI-powered business builder.

## Weekly Rhythm

| Day | Series |
| --- | --- |
| Monday | Marketing Beyond Marketing |
| Tuesday | Build With Vikash |
| Wednesday | AGMP With Vikash |
| Thursday | Mobility With Vikash |
| Friday | Buyer Psychology With Vikash |
| Saturday | Leadership, personal story, or cross-industry insight |
| Sunday | Review, engagement, and planning |

## Drafting Workflow

1. Capture the raw observation.
2. Connect it to a business context.
3. Identify the practical lesson.
4. Add a framework only if it improves clarity.
5. Write in a natural human voice.
6. Check for exaggeration, fake certainty, and generic phrasing.
7. Add series hashtag and publishing notes.

## Publishing Rule

No post should be published only because the calendar says so. The calendar creates discipline, but quality decides publication.
